# Buni 🐰

Claude Code가 실행되는 동안 화면 위에 떠있는 픽셀아트 마스코트 앱입니다.  
Claude가 생각하고, 도구를 사용하고, 권한을 요청할 때 부니가 함께 반응합니다.

---

## 설치

### macOS

**요구사항:** macOS 13 Ventura 이상 · [Claude Code](https://claude.ai/code) CLI

```bash
git clone https://github.com/EloyYang/buni.git
cd buni
bash install.sh
```

`install.sh`가 수행하는 작업:
1. Swift 빌드
2. Claude Code 훅 스크립트 설치 (`~/.claude/companion-*.py`)
3. `~/.claude/settings.json`에 훅 자동 등록
4. 앱 실행

빌드 및 재설치:
```bash
bash build_app.sh --install
```

### Windows

**요구사항:** [Claude Code](https://claude.ai/code) CLI · Python 불필요

1. [Releases 페이지](https://github.com/EloyYang/buni/releases/latest)에서 **Buni.exe** 다운로드
2. `Buni.exe` 실행
3. Claude Code 훅 설치 (최초 1회):

```bat
cd %USERPROFILE%
curl -L https://raw.githubusercontent.com/EloyYang/buni/main/windows/install_hooks.py -o install_hooks.py
python install_hooks.py
```

> **소스에서 빌드하려면:**
> ```bat
> git clone https://github.com/EloyYang/buni.git
> cd buni\windows
> install.bat
> ```

## 기능

### 상태 애니메이션
Claude Code의 상태에 따라 부니가 다르게 반응합니다.

| 상태 | 부니 동작 |
|------|-----------|
| 생각 중 (`thinking`) | 맥북 타이핑 모션 + "코딩중" 말풍선 |
| 도구 사용 중 (`toolUse`) | 맥북 타이핑 모션 + 현재 도구명 말풍선 |
| 파일 읽기 중 (`toolRead`) | 문서 들고 읽기 모션 + 서류 던지기·새 서류 받기 반복 |
| 완료 (`completed`) | 눈 크게 + 팔 올리기 + 점프 (캐릭터마다 다른 리액션) |
| 알림 (`notification`) | 깜짝 점프 + 알림 내용 말풍선 |
| 권한 요청 (`permission`) | 눈이 커지며 인터랙티브 버블 팝업 (캐릭터마다 다른 포즈) |
| 대기 (`ready`) | 가끔 아이들 애니메이션 (점프, 귀 움직임 등) |
| 유휴 (`idle`, Claude 종료) | 화면 밖으로 슬라이드 아웃 |

**맥북 타이핑 모션**: 양 팔이 안쪽으로 모이고 맥북 위에서 번갈아 움직입니다. 맥북이 팔 위에 렌더링되어 팔이 맥북에 자연스럽게 가려집니다.

**문서 읽기 모션**: 캐릭터가 서류를 들고 읽다가 주기적으로 서류를 오른쪽 위로 던지고, 새 서류가 fade-in되어 다음 내용을 읽는 모션이 반복됩니다. 왼쪽에는 서류 뭉치(바닥에 떨어진 서류 포함)가 함께 표시됩니다.

### 권한 요청 버블
Claude Code가 도구 실행 권한을 요청할 때 인터랙티브 버블이 나타납니다.

- **허용** — 해당 요청만 승인
- **전체 허용** — 이후 모든 권한 요청 자동 승인 (세션 종료까지)
- **거부** — 해당 요청 거부
- 명령어를 펼쳐서 전체 내용 확인 가능 (스크롤 지원)

### 사용량 게이지바
Claude 플랜의 5시간 슬라이딩 윈도우 사용량을 10칸 세그먼트 바로 표시합니다.

- 초록 → 노랑 → 빨강으로 색상 변화
- 사용률(%) 및 리셋까지 남은 시간 표시 (예: `↺ 4:32`)
- 5분마다 자동 갱신

### 레벨 시스템
이번 달 누적 토큰 사용량 기반으로 레벨이 올라갑니다.

- `~/.claude/projects` JSONL 파일을 스캔해 월별 input + output 토큰 합산
- 500K 토큰마다 1레벨 상승, 상한 없음
- 게이지바 위에 `★ Lv.N` 형태로 표시

### 캐릭터 선택
우클릭 컨텍스트 메뉴 → **캐릭터 변경**에서 선택할 수 있습니다.  
각 캐릭터는 고유한 색상과 아이들·완료·권한 요청 애니메이션을 가집니다.

| 캐릭터 | 특징 | 완료 리액션 | 권한 요청 포즈 |
|--------|------|-------------|----------------|
| **부니** (흰토끼) | 기본 캐릭터 | 팔 올리기 + 점프 | 눈 크게 + 당근 들기 |
| **두니** (갈색토끼) | 두꺼운 눈썹, 볼 수염 | 팔 올리기 + 아래 고개 끄덕 | 눈썹 찌푸림 + 팔 올리기 |
| **푸니** (핑크토끼) | 핑크 계열 | 양 팔 올리기 + 더블 바운스 | 오른팔 들기 |
| **주니** (주황토끼) | 주황 계열 | 양 팔 올리기 + 빠른 트리플 바운스 | 양 팔 + 점프 |
| **누니** (노란토끼) | 노란 계열, 볼터치 | 양 팔 올리기 + 점프 + 별 | 눈 크게 + 별 이펙트 |
| **우니** (연두토끼) | 연두 계열 | 양 팔 올리기 + 부드러운 고개 끄덕 | 천천히 팔 올리기 |

선택한 캐릭터는 앱 재시작 후에도 유지됩니다.  
세션마다 다른 캐릭터를 지정할 수도 있습니다 (다중 세션 시 캐릭터별로 저장).

### 세션 메모
각 캐릭터 머리 위에 짧은 메모를 띄울 수 있습니다.

- 우클릭 → **메모 추가...** / **메모 편집...**
- 메모는 캐릭터 머리 위 중앙에 흰색 텍스트로 표시 (긴 내용은 자동 줄바꿈)
- **메모 지우기**로 즉시 제거
- 세션별로 독립 저장 — 여러 세션을 동시에 실행할 때 각 캐릭터가 어떤 작업 중인지 구분하는 용도로 활용 가능 (예: "프로젝트명", "브랜치명")

### 다중 세션 지원
Claude Code 세션을 여러 개 동시에 열면 세션마다 부니가 하나씩 화면에 표시됩니다.

- 슬롯 0이 최상단, 이후 아래로 8px 간격으로 쌓임
- 각 세션은 독립된 캐릭터·메모·위치를 가짐
- 세션 종료 시 해당 부니만 사라짐

### 드래그로 위치 이동
부니 캐릭터를 드래그해 화면 어디에나 배치할 수 있습니다.

- 위치는 앱 재시작 후에도 유지
- **위치 초기화**: 컨텍스트 메뉴 또는 메뉴바 아이콘에서 기본 위치(오른쪽 상단)로 복원

### 클릭 통과
말풍선이 표시되지 않는 영역은 클릭이 그대로 하위 앱으로 전달됩니다.  
부니 캐릭터와 권한 버블 영역만 마우스 이벤트를 수신합니다.

### 전역 단축키 (macOS)
권한 요청 버블이 떠있을 때 단축키로 빠르게 응답할 수 있습니다.

- **단축키 설정**: 컨텍스트 메뉴 → **단축키 설정...**
- 설정 가능한 단축키: 허용 / 거부 / 숨기기

### 메뉴바 아이콘 (macOS)
상태바 아이콘을 통해 부니를 제어할 수 있습니다.

- 부니 보이기 / 숨기기
- Claude 열기
- 위치 초기화
- 단축키 설정
- 부팅 시 자동 실행 토글
- 메뉴바 아이콘 숨기기 (이후 부니 우클릭으로만 제어)

### 자동 업데이트 알림
GitHub Releases에 새 버전이 출시되면 6시간마다 체크해 알립니다.

- 부니 말풍선 + 시스템 알림으로 새 버전 안내
- 메뉴바 클릭 → 릴리즈 페이지로 바로 이동

## Claude Code 훅 연동

부니는 Claude Code의 훅 시스템을 통해 상태를 수신합니다.

| 훅 이벤트 | 동작 |
|-----------|------|
| `PreToolUse` | 도구 이름과 종류(read/write)를 이벤트 파일에 기록 |
| `PostToolUse` | 도구 완료 이벤트 기록 |
| `Notification` | 알림 메시지 기록 |
| `Stop` | 응답 완료 이벤트 기록 |

이벤트 파일 경로:
- macOS: `/tmp/claude-companion-events-<세션ID>.jsonl`
- Windows: `%TEMP%\claude-companion-events.jsonl`

도구 종류 분류 (`toolRead` vs `toolUse`):
- **읽기 도구** (문서 읽기 모션): `Read`, `Glob`, `Grep`, `LS`, `WebFetch`, `WebSearch`, `TodoRead` 등
- **쓰기/실행 도구** (타이핑 모션): `Write`, `Edit`, `MultiEdit`, `Bash`, `TodoWrite` 등
